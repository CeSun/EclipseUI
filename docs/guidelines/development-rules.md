# 🚫 EclipseUI 开发规范

本文档记录了 EclipseUI 项目的开发规范和禁止事项，**AI 助手必须严格遵守**。

---

## ⚠️ 核心规则 - 提交前必须确认！

**AI 助手禁止在未经用户确认的情况下执行 `git commit` 或 `git push`！**

### 正确流程

```
1. 完成工作
   ↓
2. 暂停！等待用户检查 ⏸️
   ↓
3. 用户说"可以提交" / "确认"
   ↓
4. 执行 git commit && git push ✅
```

### 提交前自检清单

**每次提交前必须检查：**

- [ ] 用户已经检查过修改内容
- [ ] 用户明确说可以提交
- [ ] 没有临时文件、截图、日志（除 `~/.openclaw/workspace/` 外）
- [ ] **没有敏感信息**（API 密钥、token、密码、appSecret 等）
- [ ] 没有调试代码（`Console.WriteLine` 等）
- [ ] 已运行 `git status` 检查变更

---

## ⛔ 禁止事项

### 0. 工作目录规范

**所有临时文件必须放在工作目录，不能放在项目目录：**

| 内容 | 正确位置 | 错误位置 |
|------|----------|----------|
| 临时文件 | `~/.openclaw/workspace/` | ❌ 项目目录 |
| 日志文件 | `~/.openclaw/workspace/` | ❌ 项目目录 |
| 截图 | `~/.openclaw/workspace/` | ❌ 项目目录 |
| 临时脚本 | `~/.openclaw/workspace/` | ❌ 项目目录 |
| 测试项目 | `~/.openclaw/workspace/` | ❌ 项目目录 |

**例外：** `docs/screenshot.png` 可以放在项目目录（用于 README 展示）

---

### 1. 禁止创建临时文件

**不允许**在项目中创建任何临时文件：

- ❌ 临时测试脚本（`*.tmp`, `*.temp`, `test_*.cs`）
- ❌ 调试输出文件
- ❌ 备份文件（`*.bak`, `*.old`, `*~`）
- ❌ 自动生成的中间文件（除 `obj/`, `bin/` 外）

**正确做法：**
- 临时测试代码用完后立即删除
- 使用系统临时目录（`%TEMP%`）存储临时文件

---

### 2. 禁止保存截图

**不允许**在项目目录中保存任何截图文件：

- ❌ `*.png`, `*.jpg`, `*.gif` 等图片文件（文档必需的除外）
- ❌ 屏幕截图（`screenshot*.png`）
- ❌ 测试图片

**例外情况：**
- ✅ `docs/screenshot.png` - 用于 README 展示的官方演示截图
- ✅ 文档中必需的图片资源

**正确做法：**
- 截图发给人后，从本地删除
- 不要保留中间截图

---

### 3. 禁止创建工具项目

**不允许**在项目中创建任何工具性质的子项目：

- ❌ 测试工具（如 `SendTool`, `TestApp`, `DemoTool`）
- ❌ 辅助脚本（如 `upload.py`, `deploy.sh`）
- ❌ 调试工具

**正确做法：**
- 工具代码用完后立即删除
- 临时脚本放在系统临时目录
- 不要提交到 Git

---

### 4. 禁止保存日志文件

**不允许**在项目目录中保存任何日志文件：

- ❌ `*.log` 文件
- ❌ 调试输出文件
- ❌ 错误日志
- ❌ 性能分析日志

**正确做法：**
- 日志输出到控制台
- 使用系统日志目录
- 调试日志用完后删除

---

### 5. 禁止硬编码敏感信息

**不允许**在代码中硬编码任何敏感信息：

- ❌ API 密钥（`appSecret`, `apiKey`, `token`）
- ❌ 密码
- ❌ 私钥
- ❌ 数据库连接字符串

**正确做法：**
- 使用配置文件（添加到 `.gitignore`）
- 使用环境变量
- 使用密钥管理服务

---

### 6. 禁止提交调试代码

**不允许**提交任何调试相关的代码：

- ❌ `Console.WriteLine()` 调试输出
- ❌ 断点
- ❌ 调试日志
- ❌ 临时注释的代码块

**正确做法：**
- 提交前删除所有调试代码
- 使用条件编译（`#if DEBUG`）
- 使用正式的日志框架

---
